# 依赖注入

# 所谓依赖：

依赖与控制反转：

依赖：类 A 用到了类 B，我们就说类 A 依赖类 B。

依赖倒置原则：高层模块不应该依赖于低层模块，二者都应该依赖于抽象。应该依赖于抽象，而不是依赖细节。

控制反转是一种思想，所谓 “控制反转” 就是反转获得依赖对象的过程。

# 大致的实现原理：

可以看出，底层使用了三个字典分别存储注册信息，构造函数需要的参数，还有作用域单例对象。注册信息类包括，要生成对象的那个类、生命周期、单例周期存储用的一个对象 Obj。

登记方法 Register 中，其实就是生成注册信息类的对象，给里面三个属性复制初始化。加入到字典里。如果给函数还传入了构造函数的参数，就存到字典里，key 是这个类名。

创建方法中，先查询注册信息字典有没有 Register 过的类，找到对应的 val 中的生命周期属性，根据这个属性做对应处理。

  • 瞬时对象：不处理
  • 单例对象：先看 val 中另一个属性 Obj 有没有值，有就直接返回，没有的话等待创建后赋值给这个 Obj。
  • Scoped: 先看作用域对象的字典里有没有，有就返回，没有等待创建加入。

以后通过反射，拿到目标类的构造函数，如果有参数就用参数字典里的值赋予。最后通过 Activator.CreateInstance () 创建对象。
