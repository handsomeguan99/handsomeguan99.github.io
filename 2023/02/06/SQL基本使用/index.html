# SQLServer 的基本使用

# 三大范式

1、每个单元格里只有一个单独的数据，不能把数据杂揉到一起存入。比如加个 "," 隔开

2、所有属性都只依赖主键，比如有个属性和主键无关 (产品与订单，订单号作主键)，产品和订单号无关，产品不是先创建订单才存在。此时就要把产品分割出去做新表。

3、传递依赖。属性 A 能决定属性 B，属性 B 又能决定属性 C，这就是传递依赖。为了便于管理，可以把这 3 个属性拆为两个表，A 决定 B 一张，B 决定 C 一张。

实际情况下可以为了性能做数据冗余。不拆分数据到两张表，一般在违反范式的数据较少的情况。
